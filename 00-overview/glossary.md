# Glossary

## Introduction
This doc defines common terms so the team uses the same language.
Add new terms when they appear in requirements or design.

## Terms
| Term | Meaning |
| --- | --- |
| Device | Physical unit that sends telemetry or receives commands |
| Fleet | Group of devices managed together |
| Telemetry | Time-series data emitted by devices |
| Ingestion | Receiving and validating incoming data |
| Tenant | Logical customer boundary in the system |
| Alert | Rule-based notification triggered by conditions |
| Schema | Structure of data payloads and fields |
| SLO | Service level objective for reliability |

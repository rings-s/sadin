# Team Roles

## Introduction
This doc describes the roles involved in building and operating the platform.
It helps clarify responsibilities and decision rights.

## Admin
- Purpose: system configuration and governance
- Typical tasks: access reviews, policy updates
- Allowed: manage users, configuration, audits

## Operator
- Purpose: day-to-day platform operations
- Typical tasks: monitoring, incident triage
- Allowed: view dashboards, acknowledge alerts

## Developer
- Purpose: build and maintain services
- Typical tasks: implement features, fix bugs
- Allowed: deploy to non-prod, update code

## Support
- Purpose: help customers resolve issues
- Typical tasks: ticket triage, log review
- Allowed: read customer data with approval

## Security
- Purpose: manage risk and compliance
- Typical tasks: access audits, threat reviews
- Allowed: enforce policies and approvals

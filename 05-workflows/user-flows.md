# User Flows

## Introduction
This doc maps how users interact with the product from start to finish.
Use it to spot gaps and align UI and backend behavior.

## Canvas
Open: [[user-flows.canvas]]

## Onboarding a new tenant
1. Create tenant and admin user.
2. Configure device types and credentials.
3. Verify data ingestion and health.

## Provisioning a device
1. Register device and assign to tenant.
2. Provide credentials and protocol details.
3. Confirm first telemetry event.

## Monitoring and alerts
1. User sets alert rules.
2. System evaluates rules on incoming data.
3. User receives notification and investigates.
